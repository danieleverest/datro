Introducción¶

Este documento detalla los DATRO Consorcios «Casa Tec II».

Los datos fiscales relevantes se recuperan de la solución de contabilidad federada de DATRO. La teneduría de libros / cuentas no es una excepción en lo que respecta a las aplicaciones web progresivas (PWA) de localhost federado. Los contratos inteligentes de las sociedades autónomas descentralizadas (DAS) de DATRO, luego recopilan el último informe para determinar qué proveedores (y qué cantidades y cuándo) pagar de forma autónoma. En este momento, los datos fiscales no se recuperan de forma totalmente autónoma durante el proceso de recompilación / actualización del documento (rebuild.sh). El script personalizado (custom.sh) y la metodología de importación de datos aún necesitan más mejoras antes de que esto suceda. Este informe se compila posteriormente manualmente, lo que actualmente es intensivo ya que el informe es nuevo y las exportaciones deben ajustarse manualmente para adaptarse al formato deseado de los documentos, especialmente cuando se trata de datos y tablas.

Los ajustes a los datos exportados hasta ahora incluyen:

  1. Queremos mantener las referencias en las transacciones como una sola cadena para que la línea no se rompa para ajustarse al ancho de la columna. p.ej. Alquiler_Débito_2020/09/01

  2. Queremos agregar .. centered:: a cada entrada en el archivo csv. Esto también ayuda a la alineación vertical. Y es mejor que automático, donde algunos elementos están alineados a la izquierda, otros alineados a la derecha etc

  3. Hacemos que las entradas seleccionadas estén en negrita o cursiva (agregando 1 o 2 estrellas al principio y al final de los datos, por ejemplo, negrita, cursiva etc

  4. Cambiamos los títulos de los columbos, p. Ej. currency se convierte en` unit` y transaction amount =` volume` - Esto permite que más monedas se aprieten en el documento. Y elimina los desplazamientos de tablas horizontales en la vista html compilada.

  5. Invierta los negativos y los positivos (en todos los números). De esta manera, el saldo final se mostrará como menos si se adeudan fondos, no como un número positivo que podría considerarse un pago en exceso / crédito.

  6. Separe el saldo final en una tabla separada. De esta manera, el número de columbs puede ser menor, ya que solo necesitamos tener un columb para el «saldo» y un columb para la moneda «USD» y uno para el monto en sí, p. -36,24

  7. Formato de fecha DD-MM-YYYY > YYYY/MM/DD
